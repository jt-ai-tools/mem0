mem0 重新排序器 (Rerankers) 會針對向量搜尋結果重新評分，確保您的代理能呈現最相關的記憶。透過此中心，您可以決定重新排序何時能提供幫助、配置提供商，並微調效能。

<Info>
重新排序是以額外的延遲換取更好的精準度。建議先建立基本的搜尋機制，並在實施前後測量相關性。
</Info>

## 選擇合適的重新排序器

- **API 優先**：當您需要頂尖品質且能負擔請求成本時（例如：Cohere, Zero Entropy）。
- **自行託管**：適用於對隱私敏感、必須保留在自有硬體上的部署（例如：Sentence Transformer, Hugging Face）。
- **LLM 驅動**：當您需要專屬的評分邏輯或複雜提示詞時。
- **混合模式**：僅在進階服務流程中啟用重新排序，以控制開銷。

## 實作檢查清單

1. 確認基準搜尋的關鍵績效指標 (KPI)，以便衡量提升效果。
2. 選擇提供商並在配置中新增 `reranker` 區塊。
3. 使用接近生產環境的查詢批次來測試延遲影響。
4. 決定要全域啟用重新排序，還是透過 `rerank` 標記針對單次搜尋啟用。
